Pagamento via Referência

Gera uma referência para pagamento em entidades ou canais tradicionais

POST https://api.paypayafrica.com/v1/payment/reference
Entity ID Referência Multicaixa

Descrição

Este endpoint gera uma referência de pagamento que o cliente pode usar para pagar em caixas automáticos, balcões ou canais tradicionais. O pagamento é identificado pelo par Entity ID + Referência. Ideal para clientes que preferem pagar em dinheiro ou não têm acesso a métodos de pagamento digital.

Parâmetros da Requisição

Parâmetro Obrig. Tipo Descrição
request_no Sim String(6,32) Número único da requisição (alfanumérico e underline)
service Sim String Valor fixo: "instant_trade"
version Sim String Versão da API: "1.0"
partner_id Sim String(32) Número do merchant (obtido no portal)
charset Sim String "UTF-8"
sign Sim String Valor da assinatura RSA
sign_type Sim String "RSA"
timestamp Sim String(19) Formato "yyyy-MM-dd HH:mm:ss" (GMT+1)
format Sim String "JSON"
language Sim String "pt" ou "en"
biz_content Sim String Conteúdo de negócio CRIPTOGRAFADO

Conteúdo de Negócio (biz_content)

Parâmetro Obrig. Tipo Descrição
payer_ip Sim String IP público do comprador
sale_product_code Sim String(9) Contactar suporte PAYPAY
cashier_type Sim String "SDK"
timeout_express Não String(6) Tempo de expiração (ex: 24h, 2d). Padrão: 24h
trade_info Sim JSON Informações do produto/serviço
pay_method Sim JSON Parâmetros do método de pagamento por referência

Informações do Produto (trade_info)

Parâmetro Obrig. Tipo Descrição
out_trade_no Sim String Número do pedido do merchant
subject Sim String Nome do produto/serviço
currency Não String "AOA" (padrão)
price Sim String Preço unitário
quantity Sim String Quantidade
total_amount Sim String Valor total (price * quantity)
payee_identity Sim String Número do merchant
payee_identity_type Não String Tipo de identidade (1 - member ID, 2 - login)

Método de Pagamento (pay_method)

Parâmetro Obrig. Tipo Descrição
pay_product_code Sim String(2) Valor fixo: "31"
amount Sim String(15) Valor do pagamento (deve ser igual ao total_amount)
bank_code Sim String Valor fixo: "REF" (Referência)

import requests
import json
import base64

# Dados do biz_content
biz_content = {
    "payer_ip": "123.25.68.9",
    "sale_product_code": "050200001",
    "cashier_type": "SDK",
    "timeout_express": "24h",
    "trade_info": {
        "currency": "AOA",
        "out_trade_no": "20240320123456789",
        "payee_identity": "200001835716",
        "payee_identity_type": "1",
        "price": "10.00",
        "quantity": "1",
        "subject": "Pagamento teste",
        "total_amount": "10.00"
    },
    "pay_method": {
        "pay_product_code": "31",
        "amount": "10.00",
        "bank_code": "REF"
    }
}

# Função para criptografar (exemplo)
def encrypt_biz_content(data):
    json_str = json.dumps(data)
    return base64.b64encode(json_str.encode()).decode()

# Requisição
url = "https://api.paypayafrica.com/v1/payment/reference"

payload = {
    "request_no": "20240320123456789",
    "service": "instant_trade",
    "version": "1.0",
    "partner_id": "200001835716",
    "charset": "UTF-8",
    "sign": "N6EPWFhOOiqP7PfgEWgffszU6SCMUimc4pQv4XL...",
    "sign_type": "RSA",
    "timestamp": "2024-03-20 14:30:00",
    "format": "JSON",
    "language": "pt",
    "biz_content": encrypt_biz_content(biz_content)
}

headers = {
    'Content-Type': 'application/json'
}

try:
    response = requests.post(url, headers=headers, json=payload)
    response.raise_for_status()
    
    data = response.json()
    print("Resposta:", json.dumps(data, indent=2))
    
    if data.get('code') == 'S0001':
        # Decodificar resposta
        response_biz = json.loads(base64.b64decode(data['biz_content']))
        print(f"Status do pagamento: {response_biz['status']}")
        print(f"Número PAYPAY: {response_biz['trade_no']}")
        print(f"ENTIDADE: {response_biz['entity_id']}")
        print(f"REFERÊNCIA: {response_biz['reference_id']}")
        
        # Instruções para o cliente
        print("\n📋 Instruções de pagamento:")
        print(f"Entidade: {response_biz['entity_id']}")
        print(f"Referência: {response_biz['reference_id']}")
        print("Valor: 10.00 AOA")
        
        if response_biz['status'] == 'P':
            print("⏳ Pagamento aguardando confirmação...")
        
except requests.exceptions.RequestException as e:
    print(f"Erro na requisição: {e}")

Conteúdo de Negócio Decodificado (3.3.6 Business content of response)

biz_content após decodificação base64 
{
  "out_trade_no": "20240320123456789",
  "trade_no": "101164267909194269883",
  "status": "P",
  "entity_id": "01010",
  "reference_id": "219485482"
}

Descrição dos campos:

  • out_trade_no: Número do pedido do merchant
  • trade_no: Número da transação PAYPAY
  • status: Status do pagamento (S-Sucesso, P-Processando, F-Falha)
  • entity_id: ID da entidade para pagamento (mostrar ao cliente)
  • reference_id: Número de referência para pagamento (mostrar ao cliente)

Nota: Para pagamentos por referência, os campos trade_token e dynamic_link não são retornados.

Status Descrição Ação

S Pagamento confirmado com sucesso Liberar produto/serviço
F Falha no pagamento ou expirado Gerar nova referência se necessário

📋 Instruções para o Cliente

Após receber a resposta com entity_id e reference_id, apresente estas informações ao cliente:

🏦 Instruções para pagamento por referência:

Entidade: 01010

Referência: 219 485 482

Valor: 10.00 AOA

Como pagar:

  1. Dirija-se a um caixa automático Multicaixa
  2. Selecione a opção "Pagamento de Serviços"
  3. Digite a Entidade: 01010
  4. Digite a Referência: 219485482
  5. Digite o Valor: 10.00 AOA
  6. Confirme o pagamento

⏱️ Válido por 24 horas

📱 Fluxo do Pagamento por Referência

  1. Requisição é enviada - O sistema do merchant envia os dados do pedido
  2. Referência é gerada - A PAYPAY gera um par (Entity ID + Referência) único
  3. Cliente recebe os dados - O merchant exibe a Entidade e Referência ao cliente
  4. Cliente efetua o pagamento - Em caixas automáticos, balcões ou canais tradicionais
  5. Notificação é recebida - O merchant recebe um webhook quando o pagamento é confirmado
  6. Consulta opcional - O merchant pode consultar o status a qualquer momento

⚠️ Validações Importantes

  • amount: Deve ser igual ao total_amount do trade_info
  • bank_code: Sempre "REF" para pagamento por referência
  • pay_product_code: Sempre "31" para este método
  • timeout_express: Recomendado 24h para referências (padrão)
  • Valor mínimo: 1.00 AOA
  • Valor máximo: 1.000.000,00 AOA por transação
