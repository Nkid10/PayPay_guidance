Transferência entre Contas PAYPAY

Transfere fundos entre contas PAYPAY de forma instantânea

POST https://api.paypayafrica.com/v1/transfer/account
Instantâneo Interno PAYPAY

Descrição

Este endpoint permite transferir fundos entre contas PAYPAY de forma instantânea. As transferências são processadas em tempo real e o crédito é imediato na conta do beneficiário. O serviço é iniciado pelo merchant, transferindo fundos da sua conta PAYPAY para a conta PAYPAY de um usuário específico.

Parâmetros da Requisição

Parâmetro Obrig. Tipo Descrição
request_no Sim String(6,32) Número único da requisição (alfanumérico e underline)
service Sim String Valor fixo: "transfer_to_account"
version Sim String Versão da API: "1.0"
partner_id Sim String(32) Número do merchant (obtido no portal)
charset Sim String "UTF-8"
sign Sim String Valor da assinatura RSA
sign_type Sim String "RSA"
timestamp Sim String(19) Formato "yyyy-MM-dd HH:mm:ss" (GMT+1)
format Sim String "JSON"
language Sim String "pt" ou "en"
biz_content Sim String Conteúdo de negócio CRIPTOGRAFADO

Conteúdo de Negócio (biz_content)

Parâmetro Obrig. Tipo Descrição
out_trade_no Sim String(6,32) Número do pedido do merchant (único)
payer_identity_type Não String(2) Tipo de ID do pagador: 1 (member ID) padrão, 2 (login)
payer_identity Sim String(6,32) ID do pagador (member ID do merchant)
payee_identity_type Não String(2) Tipo de ID do beneficiário: 1 (member ID) padrão, 2 (login)
payee_identity Sim String(6,32) ID do beneficiário (member ID ou login do destinatário)
aging Sim String(1) "R" para transferência em tempo real (único valor aceito)
transfer_amount Sim String(15) Valor a transferir (2 casas decimais)
currency Sim String(3) Moeda: "AOA" (padrão)
sale_product_code Sim String(9) Contactar suporte PAYPAY
memo Não String(128) Observações da transferência

Exemplo com Python 
import requests
import json
import base64
from Crypto.PublicKey import RSA
from Crypto.Cipher import PKCS1_v1_5

# Dados do biz_content
biz_content = {
    "out_trade_no": "20240320123456789",
    "payer_identity": "200002125756",
    "payer_identity_type": "1",
    "payee_identity": "200002125777",
    "payee_identity_type": "1",
    "aging": "R",
    "transfer_amount": "5000.00",
    "currency": "AOA",
    "sale_product_code": "050200001",
    "memo": "Pagamento a usuário"
}

# Função para criptografar (exemplo)
def encrypt_biz_content(data):
    # Implementar criptografia RSA com chave pública
    # Por simplicidade, este exemplo apenas converte para base64
    json_str = json.dumps(data)
    return base64.b64encode(json_str.encode()).decode()

# Requisição
url = "https://api.paypayafrica.com/v1/transfer/account"

payload = {
    "request_no": "20240320123456789",
    "service": "transfer_to_account",
    "version": "1.0",
    "partner_id": "200001835716",
    "charset": "UTF-8",
    "sign": "N6EPWFhOOiqP7PfgEWgffszU6SCMUimc4pQv4XL...",
    "sign_type": "RSA",
    "timestamp": "2024-03-20 14:30:00",
    "format": "JSON",
    "language": "pt",
    "biz_content": encrypt_biz_content(biz_content)
}

headers = {
    'Content-Type': 'application/json'
}

try:
    response = requests.post(url, headers=headers, json=payload)
    response.raise_for_status()
    
    data = response.json()
    print("Resposta:", json.dumps(data, indent=2))
    
    if data.get('code') == 'S0001':
        # Decodificar resposta
        response_biz = json.loads(base64.b64decode(data['biz_content']))
        print(f"Status: {response_biz['status']}")
        print(f"Trade No: {response_biz['trade_no']}")
        print(f"Data/Hora: {response_biz['order_time']}")
        
except requests.exceptions.RequestException as e:
    print(f"Erro na requisição: {e}")
Exemplo com PHP 
<?php
// Dados do biz_content
$bizContent = [
    'out_trade_no' => '20240320123456789',
    'payer_identity' => '200002125756',
    'payer_identity_type' => '1',
    'payee_identity' => '200002125777',
    'payee_identity_type' => '1',
    'aging' => 'R',
    'transfer_amount' => '5000.00',
    'currency' => 'AOA',
    'sale_product_code' => '050200001',
    'memo' => 'Pagamento a usuário'
];

// Função para criptografar (exemplo)
function encryptBizContent($data) {
    $json = json_encode($data);
    return base64_encode($json);
}

$url = 'https://api.paypayafrica.com/v1/transfer/account';

$data = [
    'request_no' => '20240320123456789',
    'service' => 'transfer_to_account',
    'version' => '1.0',
    'partner_id' => '200001835716',
    'charset' => 'UTF-8',
    'sign' => 'N6EPWFhOOiqP7PfgEWgffszU6SCMUimc4pQv4XL...',
    'sign_type' => 'RSA',
    'timestamp' => '2024-03-20 14:30:00',
    'format' => 'JSON',
    'language' => 'pt',
    'biz_content' => encryptBizContent($bizContent)
];

$json_data = json_encode($data);

$ch = curl_init($url);
curl_setopt($ch, CURLOPT_CUSTOMREQUEST, "POST");
curl_setopt($ch, CURLOPT_POSTFIELDS, $json_data);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
curl_setopt($ch, CURLOPT_HTTPHEADER, [
    'Content-Type: application/json',
    'Content-Length: ' . strlen($json_data)
]);

$response = curl_exec($ch);
$http_code = curl_getinfo($ch, CURLINFO_HTTP_CODE);

if (curl_errno($ch)) {
    echo 'Erro cURL: ' . curl_error($ch);
} else {
    echo "HTTP Code: $http_code\n";
    $response_data = json_decode($response, true);
    echo "Resposta: " . json_encode($response_data, JSON_PRETTY_PRINT) . "\n";
    
    if ($response_data['code'] == 'S0001') {
        // Decodificar biz_content da resposta
        $responseBiz = json_decode(base64_decode($response_data['biz_content']), true);
        echo "Status: " . $responseBiz['status'] . "\n";
        echo "Trade No: " . $responseBiz['trade_no'] . "\n";
        echo "Data/Hora: " . $responseBiz['order_time'] . "\n";
    }
}

curl_close($ch);
?>

Validações Importantes

  • aging: Deve ser sempre "R" (real-time)
  • payer_identity e payee_identity: Não podem ser iguais
  • transfer_amount: Máximo de 15 dígitos com 2 casas decimais
  • Saldo: A conta do pagador deve ter saldo suficiente
  • Contas: Ambas as contas devem estar ativas e verificadas
