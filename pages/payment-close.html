Fechar Pagamento

Fecha/cancela um pagamento pendente, impedindo que o cliente continue o processo

POST https://api.paypayafrica.com/v1/payment/close
Cancelamento Fechamento Gestão

Descrição

Este endpoint permite fechar ou cancelar um pagamento que está pendente. Após o fechamento, o cliente não pode mais continuar ou finalizar o pagamento. Útil para quando o pedido expira, é cancelado pelo cliente, ou por qualquer outra razão o pagamento não deve mais ser processado.

Parâmetros da Requisição

Parâmetro Obrig. Tipo Descrição
request_no Sim String(6,32) Número único da requisição (alfanumérico e underline)
service Sim String Valor fixo: "trade_close"
version Sim String Versão da API: "1.0"
partner_id Sim String(32) Número do merchant (obtido no portal)
charset Sim String "UTF-8"
sign Sim String Valor da assinatura RSA
sign_type Sim String "RSA"
timestamp Sim String(19) Formato "yyyy-MM-dd HH:mm:ss" (GMT+1)
format Sim String "JSON"
language Sim String "pt" ou "en"
biz_content Sim String Conteúdo de negócio CRIPTOGRAFADO

Conteúdo de Negócio (biz_content)

Parâmetro Obrig. Tipo Descrição
out_trade_no Sim String(6,32) Número do pedido do merchant a ser fechado

Exemplo com Python 
import requests
import json
import base64
from Crypto.PublicKey import RSA
from Crypto.Cipher import PKCS1_v1_5

# Dados do biz_content
biz_content = {
    "out_trade_no": "20240320123456789"
}

# Função para criptografar (exemplo)
def encrypt_biz_content(data):
    # Implementar criptografia RSA com chave pública
    # Por simplicidade, este exemplo apenas converte para base64
    json_str = json.dumps(data)
    return base64.b64encode(json_str.encode()).decode()

# Requisição
url = "https://api.paypayafrica.com/v1/payment/close"

payload = {
    "request_no": "20240320123456789",
    "service": "trade_close",
    "version": "1.0",
    "partner_id": "200001835716",
    "charset": "UTF-8",
    "sign": "N6EPWFhOOiqP7PfgEWgffszU6SCMUimc4pQv4XL...",
    "sign_type": "RSA",
    "timestamp": "2024-03-20 14:30:00",
    "format": "JSON",
    "language": "pt",
    "biz_content": encrypt_biz_content(biz_content)
}

headers = {
    'Content-Type': 'application/json'
}

try:
    response = requests.post(url, headers=headers, json=payload)
    response.raise_for_status()
    
    data = response.json()
    print("Resposta:", json.dumps(data, indent=2))
    
    if data.get('code') == 'S0001':
        # Decodificar resposta
        response_biz = json.loads(base64.b64decode(data['biz_content']))
        print(f"Pedido fechado: {response_biz['out_trade_no']}")
        print(f"Trade No: {response_biz['trade_no']}")
        
except requests.exceptions.RequestException as e:
    print(f"Erro na requisição: {e}")
Exemplo com PHP 
<?php
// Dados do biz_content
$bizContent = [
    'out_trade_no' => '20240320123456789'
];

// Função para criptografar (exemplo)
function encryptBizContent($data) {
    $json = json_encode($data);
    return base64_encode($json);
}

$url = 'https://api.paypayafrica.com/v1/payment/close';

$data = [
    'request_no' => '20240320123456789',
    'service' => 'trade_close',
    'version' => '1.0',
    'partner_id' => '200001835716',
    'charset' => 'UTF-8',
    'sign' => 'N6EPWFhOOiqP7PfgEWgffszU6SCMUimc4pQv4XL...',
    'sign_type' => 'RSA',
    'timestamp' => '2024-03-20 14:30:00',
    'format' => 'JSON',
    'language' => 'pt',
    'biz_content' => encryptBizContent($bizContent)
];

$json_data = json_encode($data);

$ch = curl_init($url);
curl_setopt($ch, CURLOPT_CUSTOMREQUEST, "POST");
curl_setopt($ch, CURLOPT_POSTFIELDS, $json_data);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
curl_setopt($ch, CURLOPT_HTTPHEADER, [
    'Content-Type: application/json',
    'Content-Length: ' . strlen($json_data)
]);

$response = curl_exec($ch);
$http_code = curl_getinfo($ch, CURLINFO_HTTP_CODE);

if (curl_errno($ch)) {
    echo 'Erro cURL: ' . curl_error($ch);
} else {
    echo "HTTP Code: $http_code\n";
    $response_data = json_decode($response, true);
    echo "Resposta: " . json_encode($response_data, JSON_PRETTY_PRINT) . "\n";
    
    if ($response_data['code'] == 'S0001') {
        // Decodificar biz_content da resposta
        $responseBiz = json_decode(base64_decode($response_data['biz_content']), true);
        echo "Pedido fechado: " . $responseBiz['out_trade_no'] . "\n";
        echo "Trade No: " . $responseBiz['trade_no'] . "\n";
    }
}

curl_close($ch);
?>

Códigos de Status de Pagamento

Status Descrição Pode ser fechado?
WAIT_BUYER_PAY Pagamento criado, aguardando pagamento ✅ Sim
TRADE_CLOSED Pagamento já fechado ❌ Não
TRADE_SUCCESS Pagamento bem-sucedido ❌ Não
TRADE_FINISHED Pagamento finalizado/liquidado ❌ Não

Casos de Uso

  • Expiração de pedido: Quando o tempo limite do pedido é atingido
  • Cancelamento pelo cliente: Cliente desiste da compra antes de pagar
  • Estoque indisponível: Produto ficou indisponível durante o processo
  • Fraude suspeita: Suspeita de atividade fraudulenta no pedido
  • Erro no pedido: Dados do pedido estão incorretos

Notas Importantes

  • ⚠️ Apenas pedidos com status WAIT_BUYER_PAY podem ser fechados
  • ⚠️ O fechamento é irreversível - após fechado, o pedido não pode ser reaberto
  • ⏱️ O pedido também pode ser fechado automaticamente após o timeout_express
  • 📝 O request_no deve ser único para cada requisição de fechamento
