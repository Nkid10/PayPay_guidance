Pagamento via MULTICAIXA Expre

Processa pagamentos através do serviço MULTICAIXA Expre com notificação por SMS

POST https://api.paypayafrica.com/v1/payment/multicaixa
SMS Multicaixa Express

Descrição

Este endpoint cria uma solicitação de pagamento que será processada através do MULTICAIXA Expre. O cliente recebe uma notificação por SMS para autorizar o pagamento no seu dispositivo móvel. Após a autorização, o pagamento é processado e o status é atualizado.

Parâmetros da Requisição

Parâmetro Obrig. Tipo Descrição
request_no Sim String(6,32) Número único da requisição (alfanumérico e underline)
service Sim String Valor fixo: "instant_trade"
version Sim String Versão da API: "1.0"
partner_id Sim String(32) Número do merchant (obtido no portal)
charset Sim String "UTF-8"
sign Sim String Valor da assinatura RSA
sign_type Sim String "RSA"
timestamp Sim String(19) Formato "yyyy-MM-dd HH:mm:ss" (GMT+1)
format Sim String "JSON"
language Sim String "pt" ou "en"
biz_content Sim String Conteúdo de negócio CRIPTOGRAFADO

Conteúdo de Negócio (biz_content)

Parâmetro Obrig. Tipo Descrição
payer_ip Sim String IP público do comprador
sale_product_code Sim String(9) Contactar suporte PAYPAY
cashier_type Sim String "SDK"
timeout_express Não String(6) Tempo de expiração (ex: 15m, 2H). Padrão: 2H
trade_info Sim JSON Informações do produto/serviço
pay_method Sim JSON Parâmetros do método de pagamento MULTICAIXA

Informações do Produto (trade_info)

Parâmetro Obrig. Tipo Descrição
out_trade_no Sim String Número do pedido do merchant
subject Sim String Nome do produto/serviço
currency Não String "AOA" (padrão)
price Sim String Preço unitário
quantity Sim String Quantidade
total_amount Sim String Valor total (price * quantity)
payee_identity Sim String Número do merchant
payee_identity_type Não String Tipo de identidade (1 - member ID, 2 - login)

Método de Pagamento (pay_method)

Parâmetro Obrig. Tipo Descrição
pay_product_code Sim String(2) Valor fixo: "31"
amount Sim String(15) Valor do pagamento (deve ser igual ao total_amount)
bank_code Sim String Valor fixo: "MUL" (Multicaixa)
phone_num Sim String(9) Número de telemóvel do cliente registado no Multicaixa

import requests
import json
import base64

# Dados do biz_content
biz_content = {
    "payer_ip": "123.25.68.9",
    "sale_product_code": "050200001",
    "cashier_type": "SDK",
    "timeout_express": "15m",
    "trade_info": {
        "currency": "AOA",
        "out_trade_no": "20240320123456789",
        "payee_identity": "200001835716",
        "payee_identity_type": "1",
        "price": "1000.00",
        "quantity": "1",
        "subject": "Pagamento vol",
        "total_amount": "1000.00"
    },
    "pay_method": {
        "pay_product_code": "31",
        "amount": "1000.00",
        "bank_code": "MUL",
        "phone_num": "987654321"
    }
}

# Função para criptografar (exemplo)
def encrypt_biz_content(data):
    json_str = json.dumps(data)
    return base64.b64encode(json_str.encode()).decode()

# Requisição
url = "https://api.paypayafrica.com/v1/payment/multicaixa"

payload = {
    "request_no": "20240320123456789",
    "service": "instant_trade",
    "version": "1.0",
    "partner_id": "200001835716",
    "charset": "UTF-8",
    "sign": "N6EPWFhOOiqP7PfgEWgffszU6SCMUimc4pQv4XL...",
    "sign_type": "RSA",
    "timestamp": "2024-03-20 14:30:00",
    "format": "JSON",
    "language": "pt",
    "biz_content": encrypt_biz_content(biz_content)
}

headers = {
    'Content-Type': 'application/json'
}

try:
    response = requests.post(url, headers=headers, json=payload)
    response.raise_for_status()
    
    data = response.json()
    print("Resposta:", json.dumps(data, indent=2))
    
    if data.get('code') == 'S0001':
        # Decodificar resposta
        response_biz = json.loads(base64.b64decode(data['biz_content']))
        print(f"Status do pagamento: {response_biz['status']}")
        print(f"Número PAYPAY: {response_biz['trade_no']}")
        
        if response_biz['status'] == 'S':
            print("✅ Pagamento confirmado!")
        elif response_biz['status'] == 'P':
            print("⏳ Pagamento em processamento...")
        
except requests.exceptions.RequestException as e:
    print(f"Erro na requisição: {e}")

  • trade_no: Número da transação PAYPAY

Nota: Para MULTICAIXA Expre, os campos trade_token e dynamic_link não são retornados.

Status de Pagamento

Status Descrição Ação
P Pagamento em processamento - SMS enviado, aguardando autorização Aguardar notificação ou consultar status
S Pagamento confirmado com sucesso Liberar produto/serviço
F Falha no pagamento - autorização negada ou timeout Ofercer outro método de pagamento

📱 Fluxo do Pagamento MULTICAIXA Expre

  1. Cliente informa o número - O cliente fornece o número de telemóvel registado no Multicaixa
  2. Requisição é enviada - O sistema do merchant envia a requisição com os dados do pagamento
  3. SMS é enviado - O cliente recebe uma notificação SMS com os detalhes do pagamento
  4. Cliente autoriza - O cliente autoriza o pagamento no seu dispositivo móvel
  5. Notificação é recebida - O merchant recebe uma notificação webhook com o status final
  6. Consulta opcional - O merchant pode consultar o status a qualquer momento

⚠️ Validações Importantes

  • phone_num: Deve ser um número de telemóvel angolano válido (9 dígitos)
  • amount: Deve ser igual ao total_amount do trade_info
  • bank_code: Sempre "MUL" para MULTICAIXA Expre
  • pay_product_code: Sempre "31" para MULTICAIXA Expre
  • timeout_express: Recomendado entre 15-30 minutos para MULTICAIXA
